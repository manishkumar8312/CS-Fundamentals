# Context-Free Grammars (CFG)

## 1. Introduction

A **Context-Free Grammar (CFG)** is a formal grammar used to describe the syntax of programming languages and natural languages. CFGs are powerful enough to represent nested structures such as parentheses, arithmetic expressions, and programming constructs.

CFGs are a fundamental concept in **Theory of Computation**, **Compiler Design**, and **Formal Language Theory**.

A language generated by a context-free grammar is called a **Context-Free Language (CFL)**.

---

## 2. Formal Definition of a CFG

A **Context-Free Grammar** is defined as a 4-tuple:

[
G = (**V**,**╬ú**,**R**,**S**)
]

Where:

| Symbol | Meaning                                 |
| ------ | --------------------------------------- |
| **V**  | Finite set of variables (non-terminals) |
| **╬ú**  | Finite set of terminals                 |
| **R**  | Finite set of production rules          |
| **S**  | Start symbol                            |

### Explanation

1. **Variables (Non-Terminals)**
   Symbols that can be replaced using production rules.

2. **Terminals**
   Symbols that appear in the final strings of the language.

3. **Production Rules**
   Rules describing how variables can be replaced.

4. **Start Symbol**
   The variable from which derivations begin.

### Example

Grammar:

```
S Ôćĺ aSb
S Ôćĺ ╬Á
```

This grammar generates the language:

```
{ aÔü┐ bÔü┐ | n Ôëą 0 }
```

Examples of strings generated:

```
╬Á
ab
aabb
aaabbb
```

---

## 3. Components of a Context-Free Grammar

A CFG contains the following components.

### 3.1 Terminals

Terminals are symbols that **cannot be replaced further**.

Example:

```
a, b, +, *, (, )
```

They form the **actual strings of the language**.

---

### 3.2 Non-Terminals (Variables)

Non-terminals are symbols used to **represent intermediate structures** during derivation.

Example:

```
S, A, B, Expr, Term
```

---

### 3.3 Production Rules

Production rules define **how non-terminals are replaced**.

General form:

```
A Ôćĺ ╬▒
```

Where:

* **A** is a non-terminal
* **╬▒** is a string of terminals and/or non-terminals

Example:

```
E Ôćĺ E + T
E Ôćĺ T
T Ôćĺ T * F
T Ôćĺ F
F Ôćĺ (E)
F Ôćĺ id
```

This grammar represents **arithmetic expressions**.

---

### 3.4 Start Symbol

The **start symbol** is the non-terminal where the derivation begins.

Example:

```
S
```

All strings in the language are derived starting from **S**.

---

# 4. Derivations and Sentential Forms

A **derivation** is the sequence of applying production rules to generate a string.

---

## 4.1 Sentential Form

A **sentential form** is any string obtained from the start symbol during the derivation process.

It may contain:

* terminals
* non-terminals
* both

Example:

Grammar:

```
S Ôćĺ aSb | ╬Á
```

Derivation:

```
S
aSb
aaSbb
aabb
```

Sentential forms:

```
S
aSb
aaSbb
aabb
```

The **final string containing only terminals** is called a **sentence**.

---

## 4.2 Example of Derivation

Grammar:

```
S Ôćĺ aSb | ╬Á
```

Derivation for **aabb**:

```
S
Ôçĺ aSb
Ôçĺ aaSbb
Ôçĺ aabb
```

Here the grammar generates **balanced strings of a's followed by b's**.

---

# 5. Leftmost and Rightmost Derivations

When multiple non-terminals appear in a sentential form, we can choose which one to expand first.

Two standard strategies exist.

---

## 5.1 Leftmost Derivation

In a **leftmost derivation**, the **leftmost non-terminal** is always replaced first.

### Example

Grammar:

```
S Ôćĺ AB
A Ôćĺ a
B Ôćĺ b
```

Leftmost derivation of **ab**:

```
S
Ôçĺ AB
Ôçĺ aB
Ôçĺ ab
```

---

## 5.2 Rightmost Derivation

In a **rightmost derivation**, the **rightmost non-terminal** is replaced first.

### Example

```
S Ôćĺ AB
A Ôćĺ a
B Ôćĺ b
```

Rightmost derivation:

```
S
Ôçĺ AB
Ôçĺ Ab
Ôçĺ ab
```

# 6. Parse Trees

A **Parse Tree (Derivation Tree)** is a tree representation of how a string is generated from a grammar.

Properties:

1. The **root** is the start symbol.
2. Internal nodes are **non-terminals**.
3. Leaf nodes are **terminals or ╬Á**.
4. The **left-to-right traversal of leaves** gives the generated string.

---

Grammar:

```
S Ôćĺ aSb
S Ôćĺ ╬Á
```

# 7. Ambiguity in Context-Free Grammars

A grammar is **ambiguous** if **a single string has more than one parse tree** (or more than one leftmost/rightmost derivation).

### Formal Definition

A CFG **G** is ambiguous if there exists a string **w Ôłł L(G)** such that **w has two different parse trees**.

---

## Example of Ambiguous Grammar

Grammar:

```
E Ôćĺ E + E
E Ôćĺ E * E
E Ôćĺ id
```

String:

```
id + id * id
```

Two interpretations exist:

1. `(id + id) * id`
2. `id + (id * id)`

# 8. Removing Ambiguity

Ambiguity can sometimes be removed by rewriting the grammar.

Example for arithmetic expressions:

```
E Ôćĺ E + T | T
T Ôćĺ T * F | F
F Ôćĺ (E) | id
```

This grammar enforces:

* **multiplication has higher precedence**
* **left associativity**

---

# 9. Inherently Ambiguous Languages

Some languages are **inherently ambiguous**, meaning **no unambiguous grammar exists for them**.

Even if we try to modify the grammar, ambiguity cannot be removed.

---

## Example Language

```
L = { a^i b^j c^k | i = j or j = k }
```

This language is the union of:

```
L1 = { a^n b^n c^k }
L2 = { a^i b^n c^n }
```

Strings like:

```
a^n b^n c^n
```

can be derived using **two different structures**, making ambiguity unavoidable.

Thus:

```
L is inherently ambiguous
```

# 10. Applications of Context-Free Grammars

CFGs are widely used in computer science.

### 1. Compiler Design

Used to describe programming language syntax.

Example:

```
if (condition) { statements }
```

---

### 2. Parsing

CFGs are used in parsing techniques such as:

* **LL parsers**
* **LR parsers**
* **CYK parsing**

---

### 3. Natural Language Processing

CFGs help represent grammatical structures of natural languages.

---

### 4. Expression Evaluation

Used to represent mathematical expressions and operator precedence.

---
